# AR+种草码+语音交互商品讲解系统 PRD

- 文档版本：V1.0
- 文档日期：2026-04-18
- 产品阶段：MVP / 可复制交付版
- 当前方案：方案1 `检测优先方案`
- 下一阶段预留：方案2 `物体级 AR 方案`

## 1. 项目概述

### 1.1 项目背景

本项目面向商品展览、展台导购、品牌路演等场景，目标是在固定笔记本电脑摄像头前，通过对商品的实时识别，为用户提供带有 AR 感的数字讲解体验。

用户将商品拿到摄像头前后，系统自动识别商品并与本地商品库匹配；匹配成功后，在商品附近浮现种草码（QR 码）和引导提示。用户长按种草码或商品区域后，系统在商品旁弹出信息介绍窗口，展示该商品的核心卖点、基础参数和讲解内容；同时支持用户通过语音提问，由本地语音识别和本地问答引擎返回答案，并展示在介绍窗口内。

第一版重点不是追求强空间锚定，而是优先验证一条可复制交付链路：

`商品录入 -> 本地识别 -> 内容匹配 -> 信息讲解 -> 语音问答 -> 换 SKU 复用`

### 1.2 产品定位

这是一个基于 Web 技术栈、运行于固定笔记本电脑摄像头场景下的离线商品 AR 讲解系统。  
MVP 通过“检测优先”的方式模拟 AR 讲解体验，先实现稳定识别、稳定匹配、稳定讲解，再为下一阶段的物体级 AR 升级留出接口和架构空间。

### 1.3 目标用户

- 展会现场观众
- 品牌展台导购人员
- 讲解员或主持演示人员
- 后续负责更换商品内容的运营/实施人员

## 2. 目标与非目标

### 2.1 MVP 目标

本期 MVP 必须达成以下目标：

1. 在固定笔记本摄像头下，支持 `10 个以内商品 SKU` 的离线识别与匹配。
2. 用户将商品拿到镜头前后，系统可在 `3 秒内` 完成主商品识别并显示交互入口。
3. 系统可在识别到的商品区域附近稳定显示种草码（QR 码）和引导提示。
4. 用户长按种草码或商品区域后，可弹出介绍窗口并展示本地商品内容。
5. 用户可通过语音提问获得围绕当前商品的问答回复，且全链路离线可用。
6. 新增或替换商品时，不需要改动主程序逻辑，只需更新本地商品配置、素材和识别库。

### 2.2 成功标准

本项目首要成功指标为 `可复制交付`，具体定义如下：

- 新接入一批同类型商品时，可在 `1 个工作日内` 完成内容录入与识别配置替换。
- 对同一套系统，可在不同展台或不同场地复用，无需依赖外网。
- 非研发角色按照操作文档可完成商品信息维护与本地内容替换。

### 2.3 非目标

本期不纳入范围：

- 移动端浏览器兼容优化
- 在线购买/跳转电商页
- 云端大模型或云端语音服务依赖
- 面向开放世界任意商品的通用识别
- 多商品同时精细对比讲解
- 真正的 6DoF 空间锚定和高精度 3D 姿态跟踪

## 3. 场景定义

### 3.1 典型使用场景

在展会或展台场景中，一台笔记本电脑放置在展台上，摄像头保持固定开启。用户拿起一个预置商品，在镜头前展示。系统实时识别商品并在视频画面中显示该商品的种草码和引导提示。用户长按种草码或商品本身区域，系统弹出商品介绍窗口，展示卖点、参数和讲解内容。用户继续发起语音提问，如“这个产品适合什么人群”“和上一代区别是什么”，系统在介绍窗口中返回答案。

### 3.2 场景边界

- 摄像头固定，不考虑镜头大幅移动
- 用户手持商品在镜头前移动、旋转、靠近、远离
- 商品为预先录入的标准商品，包装或外观具备一定辨识特征
- 商品数量少，适合定向建库和定向讲解

## 4. 方案选择

### 4.1 本期采用方案

本期采用 `方案1：检测优先方案`。

方案定义：

- 使用 YOLO26 预训练模型结合定向商品样本进行本地识别
- 基于检测框和目标跟踪结果，在屏幕空间中显示种草码和信息窗
- 通过视觉跟随实现“贴近商品”的 AR 感，但不承诺真实世界空间锚定
- 介绍窗口和 QR 码在商品附近随检测框运动
- 当商品离开镜头或丢失跟踪时，介绍窗口与 QR 自动消失

选择原因：

- 更符合 MVP 对稳定性和可复制交付的要求
- 在固定镜头条件下可快速实现稳定体验
- 实施成本和风险明显低于物体级姿态估计方案
- 可为后续 AR 升级保留数据结构、渲染层和跟踪接口

### 4.2 下一阶段升级方案

下一阶段预留 `方案2：物体级 AR 方案`。

升级方向：

- 在 YOLO26 初始识别基础上，引入关键点检测、特征匹配、姿态估计与多帧跟踪
- 将当前屏幕空间信息窗升级为与商品姿态关联的 3D 信息卡和 3D 框
- 在商品旋转时呈现不同角度的视觉效果
- 提高信息窗与商品表面的相对稳定性，增强“AR 眼镜式贴附”体验

本期架构要求：

- 渲染层不得与识别层强耦合
- 商品数据模型需支持后续增加 `3D 资源`、`关键点模板`、`姿态参数`
- 检测跟踪结果结构需预留 `pose`、`confidence_history`、`tracking_id`

## 5. 用户角色

### 5.1 终端用户

现场观众或客户，直接与系统交互，完成商品展示、查看讲解、语音提问。

### 5.2 现场演示人员

负责引导观众操作，必要时辅助调整商品角度、距离和交互节奏。

### 5.3 内容运营/实施人员

负责商品资料整理、录入、审核、替换本地资源。

## 6. 核心用户流程

### 6.1 主流程

1. 用户进入系统主界面。
2. 系统请求并启动笔记本摄像头。
3. 用户将商品拿到摄像头前。
4. 系统识别商品并与本地商品库匹配。
5. 匹配成功后，在商品附近显示种草码和短提示文案。
6. 用户长按种草码或商品区域。
7. 系统在商品附近弹出介绍窗口。
8. 用户查看商品信息，或点击语音提问按钮开始说话。
9. 系统完成本地语音识别，并结合本地知识库生成回答。
10. 回答在介绍窗口内展示。
11. 商品离开镜头或跟踪丢失后，种草码和介绍窗口自动消失，系统回到待识别状态。

### 6.2 异常流程

- 若未识别到商品：显示“请将商品完整放入镜头中”的提示。
- 若识别置信度低：显示“正在确认商品，请保持稳定”的提示。
- 若匹配不到商品库：显示“未录入该商品”的提示，不展示种草码。
- 若语音未识别清楚：提示用户重新提问。
- 若问答内容超出知识库：返回保守回答，如“当前资料中未包含该信息”。

## 7. 功能需求

### 7.1 首页与待机页

#### 功能描述

系统启动后进入摄像头预览主页面，界面简洁，突出实时视频画面与操作引导。

#### 需求点

- 自动尝试连接默认摄像头
- 摄像头连接成功后显示实时画面
- 页面展示简短引导文案：
  - “请将商品放到镜头前”
  - “识别成功后长按种草码查看详情”
- 提供麦克风和摄像头权限状态提示

#### 验收标准

- 页面加载后 `5 秒内` 可完成摄像头启动
- 权限异常时可见清晰错误提示

### 7.2 商品识别

#### 功能描述

系统持续从摄像头视频流中取帧，在本地调用 YOLO26 模型执行商品检测，并输出候选商品类别、置信度、检测框。

#### 需求点

- 支持本地模型加载
- 支持预置商品类目映射
- 支持最小置信度阈值配置
- 支持连续多帧确认，避免单帧抖动触发
- 支持简单目标跟踪，降低检测框跳变

#### 验收标准

- 对预置商品在正常光线下的 Top1 识别准确率达到 `85% 以上`
- 检测结果在连续画面中不能明显闪烁
- 主商品进入画面后 `3 秒内` 应出现稳定识别结果

### 7.3 商品匹配

#### 功能描述

识别结果需与本地商品库映射，若识别类别存在对应商品，则进入“可交互状态”。

#### 需求点

- 类别 ID 到商品 ID 的本地映射表
- 允许一个检测类别映射多个商品别名
- 支持商品上下架状态
- 支持本地热更新配置文件后重新加载

#### 验收标准

- 配置文件修改后无需修改主程序逻辑
- 未录入商品不得误展示旧商品内容

### 7.4 种草码展示

#### 功能描述

识别成功后，在检测框附近显示种草码和提示文案，作为后续介绍窗的显式入口。

#### 需求点

- 种草码位置默认显示在检测框右上或右侧
- 当靠近边缘时自动避让，避免遮挡和出画
- 种草码可根据商品 ID 生成或读取预制资源
- 支持长按触发，不采用单击触发，避免误触

#### 验收标准

- 种草码在检测框移动时能同步跟随
- 种草码不应大幅抖动
- 长按触发成功率高于 `95%`

### 7.5 商品介绍窗

#### 功能描述

用户长按种草码或商品区域后，系统在商品旁展示介绍窗，内容来自本地商品知识库。

#### 需求点

- 展示商品名称
- 展示一句话核心卖点
- 展示 3 到 5 条重点卖点
- 展示基础参数
- 展示适用人群/应用场景
- 支持关闭、收起、重新展开
- 支持在商品离场时自动关闭

#### 验收标准

- 介绍窗弹出时间小于 `500ms`
- 介绍内容字段完整时不得缺漏
- 商品离场后 `1 秒内` 自动收起

### 7.6 语音问答

#### 功能描述

用户在介绍窗中点击语音按钮后开始录音，系统通过本地 Whisper 模型或等价离线语音识别能力将语音转写为文本，再结合本地知识库和本地问答逻辑返回答案。

#### 需求点

- 支持点击开始录音、点击结束录音
- 支持录音状态提示
- 支持本地语音转文本
- 支持优先根据知识库字段进行回答
- 支持缺失信息时给出保守补充回答
- 回答需明确与当前商品绑定，不串商品

#### 回答策略

采用 `混合模式`：

1. 优先检索当前商品本地知识库字段
2. 若字段足够，按规则模板或本地模型生成回答
3. 若信息不足，返回“当前本地资料未覆盖该问题”的保守表述

#### 验收标准

- 普通话短句识别准确率达到可演示水平
- 回答生成时间小于 `3 秒`
- 不得回答其他商品的信息

### 7.7 商品离场与状态清理

#### 功能描述

当商品移出镜头或持续丢失检测时，系统需自动清理相关 UI 状态，避免脏状态残留。

#### 需求点

- 连续 N 帧丢失后判定商品离场
- 清理种草码
- 清理介绍窗
- 清理当前语音上下文
- 恢复到待识别状态

#### 验收标准

- 商品离场后不应继续显示旧内容
- 新商品进入后可正常重新识别与匹配

## 8. 交互状态机

系统状态定义如下：

- `Idle`：待识别状态
- `Detecting`：正在识别中
- `Matched`：已识别且已匹配商品
- `QRCodeVisible`：种草码可见
- `InfoPanelOpen`：介绍窗已打开
- `VoiceRecording`：正在录音
- `VoiceProcessing`：正在转写与回答
- `AnswerDisplayed`：答案已展示
- `Lost`：商品丢失，等待清理

核心状态流转：

`Idle -> Detecting -> Matched -> QRCodeVisible -> InfoPanelOpen -> VoiceRecording -> VoiceProcessing -> AnswerDisplayed -> Lost -> Idle`

## 10. 内容生产与运营流程

### 10.1 内容准备要求

由于当前仅有基础资料，系统需定义统一录入规范。

每个商品必须准备：

- 商品名称
- 一句话介绍
- 3 到 5 条核心卖点
- 基础参数表
- 适用场景
- 常见问题至少 5 条
- 种草码资源

### 10.2 录入流程

1. 运营整理商品资料
2. 按统一模板录入本地 JSON 或本地数据库
3. 审核字段完整性
4. 生成或替换对应种草码资源
5. 本地联调验证识别与内容匹配

### 10.3 运营目标

- 更换商品时不改前端业务逻辑
- 更换内容时不重新训练整套系统

### 11.1 架构原则

- 全链路离线运行
- Web 技术栈优先
- 识别层、匹配层、内容层、渲染层解耦
- 对方案2升级保持兼容

### 11.2 模块划分

#### 内容层

## 12. 非功能需求

### 12.1 性能

- 主画面保持基本流畅
- 识别响应时间小于 `3 秒`
- 介绍窗弹出延迟小于 `500ms`
- 语音回答延迟小于 `3 秒`

### 12.2 稳定性

- 连续运行 `4 小时` 不崩溃
- 摄像头断开后可重试恢复
- 内容文件异常时可提示错误但不导致主页面崩溃

### 12.3 可维护性

- 商品配置文件与代码分离
- 模型参数和阈值可配置
- 支持日志导出，便于现场排查

## 13. 风险与对策

### 风险1：相似商品误识别

对策：

- 首批 SKU 控制在 10 个以内
- 通过多帧确认和阈值配置降低误判
- 对外观高度相似商品避免同批录入

### 风险2：画面抖动导致 UI 不稳

对策：

- 引入简单目标跟踪和平滑策略
- 对检测框坐标做滤波

### 风险3：语音问答跑偏

对策：

- 先知识库后生成
- 未覆盖信息保守回答
- 当前上下文只绑定单商品

### 风险4：用户误以为是强空间 AR

对策：

- 在产品描述中明确为“AR 感讲解体验”
- 不承诺真实世界空间锚定
- 在下一阶段路线图中明确物体级 AR 升级方向

## 14. 验收标准

MVP 验收通过需同时满足：

1. 10 个以内预置商品可完成离线识别与内容匹配。
2. 至少 8 个商品在标准光照下可稳定识别。
3. 种草码可在识别成功后稳定显示。
4. 用户可通过长按打开介绍窗。
5. 每个商品可完成至少 5 个预置 FAQ 的问答。
6. 商品离场后 UI 可自动清理。
7. 更换一组商品资料时，无需改主程序业务逻辑。

当 MVP 验证完成后，进入方案2升级：

1. 引入商品关键点模板和姿态估计
2. 将当前 2D 跟随式信息窗升级为 3D 信息卡
3. 根据商品旋转方向展示不同角度的 3D 框
4. 增强种草码与信息卡相对商品表面的稳定性
5. 支持更自然的“AR 眼镜式讲解效果”

升级前置条件：

- 方案1 在目标场地已稳定运行
- 已沉淀标准商品录入模板
- 已形成识别配置与内容配置的复用流程

## 17. 结论

本期产品应明确定位为：

`离线、固定镜头、检测优先、可复制交付的 AR 感商品讲解系统`

不在第一版强求高精度真 AR，而是优先完成识别、交互、讲解、语音问答和内容复用闭环。  
在此基础上，下一阶段再升级为物体级 AR 方案，实现更强的空间贴附感和 3D 视觉效果。
